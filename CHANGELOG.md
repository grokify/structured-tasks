## [v0.3.0] - 2026-02-09

### Highlights

- Breaking: JSON schema migrates from snake_case to camelCase for consistency with structured-changelog
- Improved status table: phases renumbered from 1, completed phases hidden, Phase column first

### Breaking

- JSON schema field names changed from snake_case to camelCase (e.g., `ir_version` → `irVersion`, `depends_on` → `dependsOn`) ([`f6c91a4`](https://github.com/grokify/structured-tasks/commit/f6c91a4))

### Added

- Add improvements for AI agent usage including status display ([`e6d96fa`](https://github.com/grokify/structured-tasks/commit/e6d96fa))

### Changed

- Update structured-changelog dependency to v0.10.0 ([`7e44664`](https://github.com/grokify/structured-tasks/commit/7e44664))
- Renumber phases in status table starting from 1 (was 0-indexed) ([`0b608a9`](https://github.com/grokify/structured-tasks/commit/0b608a9))
- Move Phase column to first position in status table ([`7b87e95`](https://github.com/grokify/structured-tasks/commit/7b87e95))

### Fixed

- Hide completed phases from status table for cleaner output ([`fbe6b71`](https://github.com/grokify/structured-tasks/commit/fbe6b71))

### Documentation

- Migrate TASKS.json examples to camelCase and regenerate TASKS.md ([`d5f073b`](https://github.com/grokify/structured-tasks/commit/d5f073b))
- Update TASKS.json to new schema and regenerate TASKS.md ([`8f2a107`](https://github.com/grokify/structured-tasks/commit/8f2a107))
